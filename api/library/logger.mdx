## Overview

The logger module provides configured logger instances using [consola](https://github.com/unjs/consola) for consistent logging throughout the application. It includes package name tagging and log level control.

## Exports

### logger

Application logger instance with package name tag.

```typescript
export const logger: ConsolaInstance
```

The logger provides methods for different log levels:

<ResponseField name="logger.silent()" type="function">
  Silent logging (level 0)
</ResponseField>
<ResponseField name="logger.warn()" type="function">
  Warning messages (level 1)
</ResponseField>
<ResponseField name="logger.log()" type="function">
  General log messages (level 2)
</ResponseField>
<ResponseField name="logger.info()" type="function">
  Informational messages (level 3)
</ResponseField>
<ResponseField name="logger.debug()" type="function">
  Debug messages (level 4)
</ResponseField>
<ResponseField name="logger.trace()" type="function">
  Trace messages (level 5)
</ResponseField>

<CodeGroup>
```typescript Basic Usage
import { logger } from 'ccusage/logger';

logger.info('Loading usage data...');
logger.debug('Processing file:', filePath);
logger.warn('Missing cost data, using fallback');
logger.error('Failed to load file:', error);
```

```typescript With Structured Data
import { logger } from 'ccusage/logger';

logger.info('Daily usage loaded', {
  days: dailyData.length,
  totalCost: totals.totalCost
});

logger.debug('Session info', {
  sessionId: session.sessionId,
  projectPath: session.projectPath,
  cost: session.totalCost
});
```

```typescript Error Handling
import { logger } from 'ccusage/logger';

try {
  const data = await loadDailyUsageData();
  logger.info('Data loaded successfully');
} catch (error) {
  logger.error('Failed to load data:', error);
  throw error;
}
```
</CodeGroup>

### log

Direct console.log function for cases where logger formatting is not desired.

```typescript
export const log: typeof console.log
```

<Tip>
Use `log` when you need plain output without logger formatting, such as for JSON output or raw data display.
</Tip>

<CodeGroup>
```typescript JSON Output
import { log } from 'ccusage/logger';
import { loadDailyUsageData } from 'ccusage/data-loader';

const data = await loadDailyUsageData();

// Output raw JSON without logger formatting
log(JSON.stringify(data, null, 2));
```

```typescript Table Output
import { log } from 'ccusage/logger';
import Table from 'cli-table3';

const table = new Table({
  head: ['Date', 'Cost', 'Tokens'],
  style: { head: ['cyan'] }
});

table.push(['2024-01-01', '$0.15', '1000']);
table.push(['2024-01-02', '$0.23', '1500']);

// Output table without logger formatting
log(table.toString());
```
</CodeGroup>

## Log Levels

Control logging verbosity with the `LOG_LEVEL` environment variable:

| Level | Value | Methods Shown |
|-------|-------|---------------|
| Silent | 0 | None |
| Warn | 1 | `warn`, `error`, `fatal` |
| Log | 2 | `log` + above |
| Info | 3 | `info` + above (default) |
| Debug | 4 | `debug` + above |
| Trace | 5 | `trace` + all |

<CodeGroup>
```bash Environment Variable
# Silent mode (no logs)
LOG_LEVEL=0 ccusage daily

# Warnings only
LOG_LEVEL=1 ccusage daily

# Debug mode
LOG_LEVEL=4 ccusage daily

# Trace mode (maximum verbosity)
LOG_LEVEL=5 ccusage daily
```

```typescript Programmatic Control
import { logger } from 'ccusage/logger';

// Set log level programmatically
logger.level = 4; // Debug level

logger.debug('This will be shown');
logger.trace('This will not be shown');
```
</CodeGroup>

## Usage Patterns

### Development vs Production

<CodeGroup>
```typescript Development
import { logger } from 'ccusage/logger';

// Verbose logging for development
logger.debug('Starting data load...');
logger.debug('Found files:', files.length);
logger.debug('Processing file:', file);
logger.info('Data loaded successfully');
```

```typescript Production
import { logger } from 'ccusage/logger';

// Minimal logging for production
logger.info('Data load started');
logger.warn('Using offline mode');
logger.error('Data load failed:', error);
```
</CodeGroup>

### Progress Tracking

<CodeGroup>
```typescript File Processing
import { logger } from 'ccusage/logger';

const files = await globUsageFiles(claudePaths);
logger.info(`Processing ${files.length} files...`);

for (let i = 0; i < files.length; i++) {
  logger.debug(`Processing file ${i + 1}/${files.length}:`, files[i].file);
  await processFile(files[i]);
}

logger.info('All files processed successfully');
```

```typescript Data Aggregation
import { logger } from 'ccusage/logger';
import { loadDailyUsageData } from 'ccusage/data-loader';
import { calculateTotals } from 'ccusage/calculate-cost';

logger.info('Loading daily usage data...');
const dailyData = await loadDailyUsageData();

logger.debug(`Loaded ${dailyData.length} days of data`);

const totals = calculateTotals(dailyData);
logger.info('Calculation complete', {
  days: dailyData.length,
  totalCost: totals.totalCost,
  totalTokens: totals.inputTokens + totals.outputTokens
});
```
</CodeGroup>

### Error Context

<CodeGroup>
```typescript Detailed Error Info
import { logger } from 'ccusage/logger';

try {
  const data = await loadSessionUsageById(sessionId);
  if (!data) {
    logger.warn('Session not found:', sessionId);
    return null;
  }
  return data;
} catch (error) {
  logger.error('Failed to load session:', {
    sessionId,
    error: error instanceof Error ? error.message : String(error)
  });
  throw error;
}
```

```typescript Graceful Degradation
import { logger } from 'ccusage/logger';
import { calculateContextTokens } from 'ccusage/data-loader';

const context = await calculateContextTokens(transcriptPath, modelId);

if (!context) {
  logger.warn('Context tokens unavailable, using estimate');
  // Fallback to estimate
  return estimateContextTokens(data);
}

logger.debug('Context tokens calculated:', context);
return context;
```
</CodeGroup>

## Integration Examples

### CLI Commands

<CodeGroup>
```typescript Daily Command
import { logger, log } from 'ccusage/logger';
import { loadDailyUsageData } from 'ccusage/data-loader';

export async function dailyCommand(options: Options) {
  logger.info('Loading daily usage data...');
  
  const data = await loadDailyUsageData({
    since: options.since,
    until: options.until,
    mode: options.mode
  });
  
  logger.debug(`Loaded ${data.length} days`);
  
  if (options.json) {
    // Use log for raw JSON output
    log(JSON.stringify(data, null, 2));
  } else {
    // Use logger for formatted output
    logger.info('Daily Usage Report');
    // ... render table ...
  }
}
```
</CodeGroup>

### Library Usage

<CodeGroup>
```typescript Custom Analysis
import { logger } from 'ccusage/logger';
import { loadDailyUsageData } from 'ccusage/data-loader';
import { calculateTotals } from 'ccusage/calculate-cost';

export async function analyzeUsage(projectName: string) {
  logger.info('Starting usage analysis for:', projectName);
  
  const data = await loadDailyUsageData({
    project: projectName,
    groupByProject: true
  });
  
  if (data.length === 0) {
    logger.warn('No usage data found for project:', projectName);
    return null;
  }
  
  logger.debug('Data loaded:', {
    days: data.length,
    firstDate: data[0]?.date,
    lastDate: data[data.length - 1]?.date
  });
  
  const totals = calculateTotals(data);
  
  logger.info('Analysis complete:', {
    project: projectName,
    totalCost: totals.totalCost,
    avgCostPerDay: totals.totalCost / data.length
  });
  
  return { data, totals };
}
```
</CodeGroup>

## Best Practices

<Warning>
Never use `console.log` directly in ccusage code. Always use `logger` for formatted output or `log` for raw output.
</Warning>

### When to Use Each Level

- **`trace`**: Very detailed debugging, variable values
- **`debug`**: Detailed flow information, file processing
- **`info`**: General progress updates, completion messages
- **`log`**: Standard output, user-facing messages
- **`warn`**: Non-critical issues, fallback behaviors
- **`error`**: Errors that prevent operation

### Logger vs Log

- **Use `logger`**: For application messages, progress updates, errors
- **Use `log`**: For JSON output, tables, raw data that shouldn't be formatted

<CodeGroup>
```typescript Good: Using logger for messages
import { logger } from 'ccusage/logger';

logger.info('Processing complete');
logger.debug('Found', items.length, 'items');
logger.warn('Missing optional field:', field);
```

```typescript Good: Using log for raw output
import { log } from 'ccusage/logger';

log(JSON.stringify(data));
log(table.toString());
log(rawOutput);
```

```typescript Bad: Using console.log
// ❌ Don't do this
console.log('Processing complete');
console.log(JSON.stringify(data));
```
</CodeGroup>
